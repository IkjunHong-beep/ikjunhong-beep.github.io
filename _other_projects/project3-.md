**Challenge**- Large language models (LLMs) suffer from hallucinations and unreliable outputs, especially in domain-specific tasks. Existing solutions often increase computational cost and system complexity.

**Approach**- Developed a lightweight RAG (Retrieval-Augmented Generation) pipeline to ground LLM responses with relevant external knowledge, Designed an evaluation framework to quantify hallucination reduction and response quality.

**Techniques**- LLM agents, RAG pipelines (embedding + vector search), Hybrid retrieval (keyword + embedding), Evaluation metrics (precision, recall, LLM-as-a-judge)  

**Instruments**- Python, FAISS/ChromaDB, Sentence-transformers, LLM APIs
